# 3장

## 책에서 제시된 처리율 제한 부분 의문

* 책에서는 하나의 로드밸런서가 여러 다중화된 웹 서버로 요청을 전달한다.

처리율 제한을 계산할 때 처리율을 관리하는 포인트는 어디가 되는 것이 보통적일까?

웹 서버에서 처리율 제한을 계산한다면 기존의 요청을 세는 부분이 존재해야할 것이고 이는 각 서버 인스턴스마다 다르면 곤란할 것이라고 생각했다.

조사된 부분

* API Gateway나 로드밸런서도 결국 다중화된다. 즉 이들도 처리율에 관한 데이터를 얻기 위해 Redis와 같은 것을 사용해야한다.

* 웹 서버에서 처리할 경우, 좀 더 자세한 분기가 가능하다. 예를 들어 사용자ID, 멤버십 등급, 특정 비즈니스 로직에 따라 세밀하게 제한을 걸 수 있다.

* 내부 서버까지 들어오지 않게 하는 것은 DDoS 방어에선 중요한 포인트이다.
