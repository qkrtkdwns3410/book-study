## 모든 노드마다 색인이 된다면, 경우의 수가 미친듯이 늘어 날텐데 저장공간이 감당할 수 있을까?
G라는 단어 자체를 기준으로 Top-K frequency가 결정되는 것이 아니라, 완성된 검색어(예: GTX 1080)를 기준으로 전체 ranking이 계산된다.

즉, 특정 prefix(G)에 대해 별도로 순위를 계산하는 것이 아니라,
이미 집계된 검색어의 frequency를 기반으로 해당 검색어가 포함되는 모든 prefix에 전파된다.

예를 들어,
GTX 1080의 검색 빈도가 높아 ranking이 높게 결정되었다면,

- g
- gt
- gtx
- gtx 1

과 같이 해당 검색어가 포함되는 모든 prefix 노드에 대해,
Top-K 후보로 반영되도록 업데이트된다.

그렇다면 이러한 frequency는 어떤 기준으로 순위가 결정될까?

단순히 검색 횟수만으로 정렬하는 것이 아니라,
일반적으로는 검색 횟수에 시간, 트렌드, 클릭률 등의 요소를 결합하여 score를 산정한다.

예를 들어, NVIDIA에서 GTX 5090을 발표했다고 가정하면,
해당 키워드에 대한 검색량이 급증하고, 사용자 클릭 또한 증가하게 된다.
이러한 변화는 트렌드 및 사용자 반응 지표로 반영되어, 해당 검색어의 score를 빠르게 상승시키는 요인으로 작용한다.

이처럼 계산된 score를 기반으로 자동완성 및 검색 결과의 우선순위가 결정된다.

다만, 이러한 score를 사용자 입력마다 실시간으로 재계산하여 색인에 반영하는 것은 비용이 크기 때문에,
실제 시스템에서는 로그 데이터를 수집한 뒤 배치 처리 또는 스트리밍 기반 집계를 통해 일정 주기로 score를 갱신하고, 
이를 자동완성 인덱스에 반영하는 방식을 사용한다.


## 검색 시 어느 간격으로 API 호출이 이루어지는가?

사용자의 키보드 입력마다 즉시 API를 호출할 경우, 서버 부하가 과도하게 증가하고 불필요한 네트워크 요청이 발생한다.  

호출을 제어하는 방법

### 2. Debounce

입력이 멈춘 후 일정 시간 뒤에 API를 호출하는 방식이다.

예를 들어,  
사용자가 `g → gt → gtx`를 빠르게 입력할 경우, 매 입력마다 호출하지 않고  
마지막 입력 이후 200~300ms 동안 추가 입력이 없을 때 한 번만 호출한다.

이를 통해 불필요한 호출을 크게 줄일 수 있다.


### 3. 최소 입력 길이 제한

일정 길이 이상의 입력에서만 API를 호출하도록 제한한다.

- 1글자: 호출하지 않음  
- 2글자 이상: 호출

이유는 다음과 같다.

- `g`와 같은 단일 문자 입력은 후보가 지나치게 많음  
- 결과의 의미가 낮고, 서버 부하 대비 효율이 떨어짐 
  - 근데 구굴같은 검색엔진이 기반인 플랫폼은 당연히 한글자 또한 자동완성 가능하게 설계함.


### 4. 이전 요청 취소

입력 도중 발생한 이전 API 요청을 취소하지 않으면, 응답 순서가 뒤바뀌는 문제가 발생할 수 있다.

예:

- `gt` 요청 전송 → 서버 응답 지연  
- `gtx` 요청 전송 → 서버 응답 빠름  

이 경우:

- `gtx` 응답이 먼저 도착  
- 이후 `gt` 응답이 도착하여 최신 결과를 덮어씀  

(즉, 이모 짜장면 하나요! > 아 짬뽕으로 바꿀게요! > 근데 짜짱면이 나옴..)
결과적으로 화면에 과거 데이터가 표시되는 문제가 발생한다.

이를 방지하기 위해, 새로운 요청이 발생하면 이전 요청을 취소해야 한다.

- iOS: URLSession cancel  
  진행 중인 네트워크 요청을 중단하여, 더 이상 해당 응답이 처리되지 않도록 한다.
- Web: AbortController  
  fetch 요청을 취소하여, 이전 요청의 응답이 도착하더라도 무시되도록 한다.


### 5. 캐시

이미 조회한 prefix에 대해서는 API를 다시 호출하지 않고 캐시된 결과를 사용한다.

예:

- "gt"를 이미 조회했다면  
→ 동일한 요청에 대해 재호출하지 않음

사용자가 타이핑이 느린 할아버지라고 가정

g.. gt ... gtx...gtx5.. 이런식으로 천천히 쳐서 모든 API 요청이 간다고 가정했을때,
할아버지가 gtx5..까지 치다가 아! 4090이었는데.. 하면서 백스페이스를 칠경우(gtx5 -> gtx)
이미 gtx의 호출은 처리되었기 때문에, 사용자 캐시에 저장해두는것이 훨씬 이득.


### 6. 설계 예시

일반적으로 다음과 같은 조건을 조합하여 사용한다.

- 최소 입력 길이: 2글자 이상  
- debounce: 약 250ms  
- 이전 요청 취소  
- 클라이언트 캐시 활용
