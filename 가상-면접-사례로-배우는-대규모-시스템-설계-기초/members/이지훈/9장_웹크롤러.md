
# 웹 크롤러 중복 탐지 알고리즘

웹 크롤러가 중복된 페이지를 모두 수집하면 성능이 떨어지고 저장 공간이 낭비된다.
이를 막기 위해 중복 콘텐츠를 자동으로 탐지하는 알고리즘이 필요하다.

---

## 1. 단순 해시 기반 중복 제거

페이지 전체 텍스트를 하나의 문자열로 보고 MD5, SHA 같은 해시를 만든다.
해시값이 같으면 완전히 동일한 문서라고 판단한다.

예시:

텍스트1: Hello world!
텍스트2: Hello world!!

문자 하나만 달라도 해시값은 완전히 달라진다.
그래서 광고나 날짜 같은 작은 차이만 있어도 다른 문서로 인식한다.

장점: 구현이 단순하다  
단점: 근사 중복을 못 잡는다

---

## 2. 근사 중복(near-duplicate) 탐지

문제는 내용이 거의 같은데 일부만 다른 경우다.
이를 위해 본문 유사도를 빠르게 계산하는 알고리즘을 사용한다.

### 2.1 Shingling + MinHash

1. 문서를 작은 조각(shingle)으로 나눈다.
   예: "abcde" → ["ab","bc","cd","de"]

2. 각 문서를 집합으로 만든 뒤 Jaccard 유사도를 계산한다.

Jaccard(A,B) = |A ∩ B| / |A ∪ B|

```text
분자(|A ∩ B|)는 두 문서에 공통으로 존재하는 shingle 개수

분모(|A ∪ B|)는 두 문서 전체 shingle의 합집합 개수

즉, 겹치는 비율을 계산하는 방식이다.

예를 들어,

A = {ab, bc, cd, de}
B = {ab, bc, cx, xd}

공통 요소는 {ab, bc} → 2개
전체 요소는 {ab, bc, cd, de, cx, xd} → 6개

Jaccard = 2 / 6 = 0.33
```
값이 1에 가까울수록 두 문서는 비슷하다.

장점: 비교적 정확하다  
단점: 계산 비용이 크다.

MinHash 알고리즘으로 계산 비용 최적화

1. MinHash를 사용하면 실제 집합을 직접 비교하지 않고도
   유사도를 빠르게 추정할 수 있다.


---

### 2.2 SimHash

SimHash는 문서를 하나의 비트 패턴(fingerprint)으로 만든다.
비슷한 문서는 비슷한 비트 패턴을 가진다.

예시:

문서1: "a b c d"
문서2: "a b x d"

1. 각 단어를 해시값으로 변환한다.
2. 비트 위치별로 가중 합을 계산한다.
3. 최종적으로 0보다 크면 1, 작으면 0으로 바꿔 하나의 비트열을 만든다.

fingerprint1: 101010101010
fingerprint2: 101010001010

두 비트열의 다른 위치 개수를 해밍 거리라고 한다.
해밍 거리가 작으면 문서 내용도 유사하다고 판단한다.

장점: 빠르고 확장성 좋다  
단점: 완벽하게 동일한 유사도 계산은 아니다

---

## 3. Bloom Filter

Bloom Filter는 이미 본 적 있는 콘텐츠인지 빠르게 판단하는 자료구조다.

- 비트 배열과 여러 해시 함수를 사용한다.
- 거짓 양성(false positive)은 가능하다.
- 거짓 음성(false negative)은 없다.

즉, "처음 본 콘텐츠"라고 나오면 확실히 처음 본 것이고,
"중복 같다"고 나오면 추가 검사로 넘긴다.

Bloom Filter는 근사 중복을 판단하는 도구가 아니라
1차 필터링 도구로 사용된다.

---

## 4. 실제 처리 흐름

1. URL 중복 체크
2. 텍스트 정규화
3. Bloom Filter 1차 필터링
4. SimHash 또는 MinHash 비교
5. 중복 아니면 저장

---

## 5. 정리

- 단순 해시는 완전 동일한 문서만 잡는다.
- 근사 중복 탐지는 SimHash, MinHash 등을 사용한다.
- Bloom Filter는 빠른 1차 필터 역할을 한다.
