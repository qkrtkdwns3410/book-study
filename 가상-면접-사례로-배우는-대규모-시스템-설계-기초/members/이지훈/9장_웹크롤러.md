
# 웹 크롤러 중복 탐지 알고리즘

웹 크롤러가 중복된 페이지를 모두 수집하면 성능이 떨어지고 저장 공간이 낭비된다.
이를 막기 위해 중복 콘텐츠를 자동으로 탐지하는 알고리즘이 필요하다.

---

## 1. 단순 해시 기반 중복 제거

페이지 전체 텍스트를 하나의 문자열로 보고 MD5, SHA 같은 해시를 만든다.
해시값이 같으면 완전히 동일한 문서라고 판단한다.

예시:

텍스트1: Hello world!
텍스트2: Hello world!!

문자 하나만 달라도 해시값은 완전히 달라진다.
그래서 광고나 날짜 같은 작은 차이만 있어도 다른 문서로 인식한다.

장점: 구현이 단순하다  
단점: 근사 중복을 못 잡는다

---

## 2. 근사 중복(near-duplicate) 탐지

문제는 내용이 거의 같은데 일부만 다른 경우다.
이를 위해 본문 유사도를 빠르게 계산하는 알고리즘을 사용한다.

### 2.1 Shingling + MinHash

1. 문서를 작은 조각(shingle)으로 나눈다.
   예: "abcde" → ["ab","bc","cd","de"]

2. 각 문서를 집합으로 만든 뒤 Jaccard 유사도를 계산한다.

Jaccard(A,B) = |A ∩ B| / |A ∪ B|

```text
분자(|A ∩ B|)는 두 문서에 공통으로 존재하는 shingle 개수

분모(|A ∪ B|)는 두 문서 전체 shingle의 합집합 개수

즉, 겹치는 비율을 계산하는 방식이다.

예를 들어,

A = {ab, bc, cd, de}
B = {ab, bc, cx, xd}

공통 요소는 {ab, bc} → 2개
전체 요소는 {ab, bc, cd, de, cx, xd} → 6개

Jaccard = 2 / 6 = 0.33
```
값이 1에 가까울수록 두 문서는 비슷하다.

장점: 비교적 정확하다  
단점: 계산 비용이 크다.

MinHash 알고리즘으로 계산 비용 최적화

1. MinHash를 사용하면 실제 집합을 직접 비교하지 않고도
   유사도를 빠르게 추정할 수 있다.


### 2.2 SimHash

SimHash는 문서를 하나의 비트 패턴(fingerprint)으로 만든다.
비슷한 문서는 비슷한 비트 패턴을 가진다.

#### 예시

SimHash는 문서를 하나의 비트 지문 으로 압축해서
두 문서의 "비슷함"을 해밍 거리(Hamming Distance)로 판단하는
방식이다.

#### 예시 문서

문서 A\
"apple banana coffee milk"

문서 B\
"apple banana tea milk"

→ 거의 비슷하고 `coffee` 만 `tea` 로 바뀌었다.


#### 1단계: 각 단어를 고정 길이 해시로 변환 (예: 4비트로 단순화)

4비트만 사용한다고 가정

  단어     해시값
  -------- --------
  apple    1010
  banana   1100
  coffee   1001
  tea      0111
  milk     1011


#### 2단계: 비트 위치별 가중 합 계산

각 비트 자리에서:

-   1이면 +1
-   0이면 -1

으로 누적한다.


#### 문서 A 계산

apple → 1 0 1 0\
banana → 1 1 0 0\
coffee → 1 0 0 1\
milk → 1 0 1 1

이걸 열(column) 기준으로 계산하면:

  위치   계산
  ------ ------------------
  1번    +1 +1 +1 +1 = +4
  2번    -1 +1 -1 -1 = -2
  3번    +1 -1 -1 +1 = 0
  4번    -1 -1 +1 +1 = 0


#### 3단계: 0보다 크거나 같으면 1, 작으면 0

문서 A fingerprint

1 0 1 1

→ fingerprint A = 1011


#### 문서 B 계산

apple → 1 0 1 0\
banana → 1 1 0 0\
tea → 0 1 1 1\
milk → 1 0 1 1

계산:

  위치   계산
  ------ ------------------
  1번    +1 +1 -1 +1 = +2
  2번    -1 +1 +1 -1 = 0
  3번    +1 -1 +1 +1 = +2
  4번    -1 -1 +1 +1 = 0

→ fingerprint B = 1011


#### 결과

문서 A: 1011\
문서 B: 1011

해밍 거리 = 0

→ 거의 동일한 문서로 판단


#### 만약 완전히 다른 문서라면?

문서 C\
"car engine oil tire"

전혀 다른 단어들이라면 비트 누적 패턴이 크게 달라진다.

예:

C fingerprint = 0100

A: 1011\
C: 0100

해밍 거리 = 4

→ 완전히 다른 문서

#### 요약하자면..

-   단어 하나 바뀌면 fingerprint도 조금만 변한다.
-   많이 바뀌면 fingerprint도 크게 변한다.
-   그래서 해밍 거리 ≤ k 같은 조건으로 근사 중복을 빠르게 판별한다.
-   실제로는 64비트 또는 128비트를 사용한다.

두 비트열의 다른 위치 개수를 해밍 거리라고 한다.
해밍 거리가 작으면 문서 내용도 유사하다고 판단한다.

장점: 빠르고 확장성 좋다  
단점: 완벽하게 동일한 유사도 계산은 아니다

## 3. Bloom Filter

Bloom Filter는 이미 본 적 있는 콘텐츠인지 빠르게 판단하는 자료구조다.

- 비트 배열과 여러 해시 함수를 사용한다.
- 거짓 양성(false positive)은 가능하다.
- 거짓 음성(false negative)은 없다.

즉, "처음 본 콘텐츠"라고 나오면 확실히 처음 본 것이고,
"중복 같다"고 나오면 추가 검사로 넘긴다.

Bloom Filter는 근사 중복을 판단하는 도구가 아니라
1차 필터링 도구로 사용된다.

---

## 4. 실제 처리 흐름

1. URL 중복 체크
2. 텍스트 정규화
3. Bloom Filter 1차 필터링
4. SimHash 또는 MinHash 비교
5. 중복 아니면 저장

---

## 5. 정리

- 단순 해시는 완전 동일한 문서만 잡는다.
- 근사 중복 탐지는 SimHash, MinHash 등을 사용한다.
- Bloom Filter는 빠른 1차 필터 역할을 한다.
