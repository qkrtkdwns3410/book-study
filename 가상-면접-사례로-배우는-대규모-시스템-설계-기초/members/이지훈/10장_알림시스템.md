# Shopify 고객 서버 마이그레이션 이후 대량 계정 활성화 메일 발송 전략

---

## 1. 개요

기존 쇼핑몰 플랫폼에서 Shopify로 서버를 이전하는 경우  
기존 고객들에게 **쇼핑몰 이전 사실 및 계정 활성화 절차 안내 메일**을 발송해야 한다.

마이그레이션 이후 고객 데이터는 이관되지만  
보안 정책상 **비밀번호는 초기화되므로 고객은 반드시 계정 활성화를 수행해야 한다.**

**15만 명 규모 고객 대상 계정 활성화 메일 발송 전략 및 구현 방식**

---

## 2. 목표

- 모든 고객에게 쇼핑몰 이전 사실 전달
- 계정 활성화가 필요한 고객만 정확히 선별하여 메일 발송
- Shopify API Rate Limit을 고려한 최대 처리 속도 확보
- 메일 발송 상태 로그 관리
- 재발송 전략 수립 가능하도록 설계

---

## 3. 전체 처리 흐름

1. 고객 데이터 Shopify로 마이그레이션
2. Shopify 내부 Customer ID 확보
3. 전체 고객 목록 조회
4. 계정 상태 조회 (활성화 여부 판단)
5. 비활성 고객만 필터링
6. 비동기 방식으로 계정 활성화 메일 발송
7. 발송 결과 CSV 저장
8. 일정 기간 후 미활성 고객 재발송

---

## 4. 고객 조회 스크립트 설계

Shopify GraphQL Pagination을 이용하여 전체 고객 ID를 수집한다.

### 사용 Query

```graphql
query GetCustomers($first: Int!, $after: String) {
  customers(first: $first, after: $after) {
    edges {
      node {
        id
        email
      }
      cursor
    }
    pageInfo {
      hasNextPage
      endCursor
    }
  }
}
````

### 구현

* Rate Limit 제어
* CSV 파일로 결과 저장 후 대량 호출 API input data 로 사용

### 주요 함수
(저희 스터디가 오픈소스라, 함수 내부는 회사 레포안에서 확인할수있습니다..ㅎㅎ)

```js
async function* fetchCustomers(url, token, pageSize, rateLimiter)
```

* Shopify 고객을 페이지 단위로 조회
* Generator 방식으로 메모리 사용 최소화
* 대량 고객 조회 시 안정적인 처리 가능

```js
const rateLimiter = createRateLimiter(args.rateLimit);
```

* 초당 API 호출 수 제어
* Shopify Rate Limit 초과 방지


## 5. 계정 활성화 메일 발송 스크립트 설계

### 사용 Mutation

```graphql
mutation customerSendAccountInviteEmail($customerId: ID!) {
  customerSendAccountInviteEmail(customerId: $customerId) {
    userErrors {
      field
      message
    }
  }
}
```

---

### 주요 설계 포인트

* CSV Chunk 단위 처리 (대용량 대응)
* Concurrency Worker Async
* Rate Limit 제어
* 실패 로그 + 결과 CSV 저장

---

### Chunk 기반 CSV 처리
```js
async function* readCsvChunks(filePath, chunkSize)
```

* 대량 CSV 파일을 한 번에 메모리에 올리지 않음
* chunkSize 단위로 처리
* 안정적인 대량 발송 가능

---

### 병렬 처리 구조

```js
// item : 작업 csv row, courrency: 워커풀 개수, handler: inviteAPI
async function runWithConcurrency(items, concurrency, handler)
```

* Worker Pool 구조
* 동시에 여러 고객에게 메일 발송 가능
* 처리 속도 극대화

---

## 6. 고객 상태 조회 전략

계정 활성화 메일은 **비활성 고객에게만 발송해야 한다.**

Shopify 고객 상태는 다음 기준으로 판단 가능하다.

* `state = ENABLED` → 이미 활성화된 고객
* `state = INVITED` 또는 비밀번호 미설정 → 활성화 필요 고객

따라서 전체 고객 조회 이후
**상태 조회 API를 통해 비활성 고객 리스트를 생성한 후 메일 발송 대상 선정**
