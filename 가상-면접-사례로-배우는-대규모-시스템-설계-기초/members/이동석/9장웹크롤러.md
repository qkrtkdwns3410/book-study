# 웹페이지의 해시값을 어떻게 빠르게 비교할까요?
- 책의 145pg 중복 콘텐츠를 구별할때 해시값을 비교하면 빠르게 중복을 확인할수 있다고 한다.

- 자카드 유사도
    - 자카드 유사도(Jaccard Similarity)는 두 개의 집합 사이의 유사도를 측정하는 지표입니다. 즉, 교집합의 크기를 합집합의 크기로 나눈 값으로 정의하며 수식으로 표현하면 다음과 같습니다.
        - ex) 문장의 유사도를 구할때는 단순히 계산하면 단어를 원소로 할수 있다.
    - 참고자료 : https://wewegh.tistory.com/95
- minhash
    - 등장 배경 :
        - **자카드 유사도를 단어 단위로 비교하여 사용하게 되면, 토큰의 개수가 엄청나게 많아지게 됨**으로써, 일일이 비교하는 것이 **시간이 많이 소요**되게 됩니다.
        - 따라서 **유사도를 근사하여 측정하는 방법이 필요하게 되고, 여기서 고안된 방법이 MinHash**

      - minhash를 자카드 유사도 측정을 위한 원소로 사용하여 자카드 유사도를 계산한다.
      - 해시함수를 이용한 일부 단어만 선정, 유사도를 측정하게 된다.
    - **특징**
        - **해시 함수의 개수가 많아지면, 실제로 측정된 유사도는 자카드 유사도와 거의 동일**해지는데, 이를 이용하여 자카드 유사도의 근사를 수행해볼 수 있습니다.
        - 너무 많은 개수의 해시 함수를 사용하게되면 수행 시간의 이점은 작아지겠지만,
        - **토큰의 개수보다 훨씬 적은 개수로도 문서 간 유사도 측정이 쉬워져** 많이 사용되는 방법입니다.

- 시간복잡도의 비교
    - **1. 단순 자카드 유사도 (Exact Jaccard)**
    두 집합 $A$와 $B$의 유사도를 직접 구할 때의 비용입니다.
    • **계산 방식:** 두 집합의 원소를 전수 조사하여 교집합과 합집합의 크기를 측정합니다.
    • **두 집합 비교 시간 복잡도:** $O(n)$
        ◦ 여기서 $n$은 집합의 원소 개수입니다. (Hash Set 등을 사용해 교집합을 찾을 때 기준)
    • **전체 데이터셋($N$개의 문서) 비교 시:** $O(N^2 \cdot n)$
        ◦ 모든 문서 쌍($N^2$)을 일일이 대조해야 하므로, 문서 수가 늘어날수록 계산량이 기하급수적으로 증가합니다.
    - **2. MinHash (Approximate Jaccard)**
    MinHash는 집합을 작은 크기의 **'서명(Signature)'**으로 압축하여 비교하는 방식입니다.
    • **계산 방식:** 각 집합에 대해 $k$개의 해시 함수를 적용하여 가장 작은 해시값들(Signatures)만 저장합니다.
    • **두 집합 비교 시간 복잡도:** $O(k)$
        ◦ $k$는 해시 함수의 개수(보통 수백 개)입니다. 원래 데이터의 크기 $n$이 아무리 커도 **고정된 길이 $k$만 비교**하면 됩니다.
    • **전체 데이터셋 비교 시:** $O(N^2 \cdot k)$
        ◦ 여전히 모든 쌍을 비교한다면 $N^2$이 남지만, 개별 비교 비용이 $n$에서 $k$로 크게 줄어듭니다.
        ◦ 여기에 **LSH(Locality Sensitive Hashing)** 기술을 결합하면 $N^2$의 복잡도를 $O(N)$에 가깝게 획기적으로 낮출 수 있습니다.
- **LSH(Locality Sensitive Hashing)**
    - Bucket Hashing은 LSH에서 핵심이 되는 기술로, **각 band 내 signature의 조합을**
    - **임의의 bucket에 hashing**하여 같은 signature여부를 간접적으로 따지는 방법을 의미합니다.
    - 예시 1 : https://jimmy-ai.tistory.com/118?pidx=0
    - 예시 2 : https://jimmy-ai.tistory.com/118?pidx=1
    - 참고 자료 : https://jimmy-ai.tistory.com/118
    - LSH에서는 두 문서가 단 1개의 band에서라도 같은 bucket으로 해싱되면, 이 두 문서는 유사한 문서로 취급하게 된다는 가정입니다.
    - ## **LSH(Locality Sensitive Hashing)라고 불리는 이유**
        * **Locality (국소성):** 데이터 간의 물리적 거리나 유사성을 보존한다는 의미입니다.
        * **Sensitive (민감한):** 데이터가 서로 유사할수록 같은 해시 값(Bucket)에 담길 확률이 **민감하게 반응하여 높아지도록** 설계되었기 때문입니다.
        * **Hashing (해싱):** 방대한 데이터를 고정된 크기의 값으로 변환하여 빠르게 검색하고 비교하는 알고리즘의 본질을 나타냅니다.
    - 장점
        - 시간 복잡도 : asis : O(N^2 ) tobe :평균적으로 O(N^{1+ρ}) (ρ < 1) 수준으로 감소
